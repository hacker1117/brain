# Henry 的四项需求优先级排序

## Henry 提出的四项需求

主公在 2026-05-13 提出以下四项需求：

1. **日常沟通开会流程整合到 GBrain** — 飞书线上会议 + Getseed 线下录音总结
2. **搞清 GBrain 里的认知如何被 Agent 在对话中调用** — 不只是存储，要能用上
3. **Telegram group 多 topic 自动回复** — 不需要 @，能调用 GBrain skill + capture 信息
4. **基于电商公司 + 培训业务的会议记录做技术/培训方案讨论**

[Source: User, Telegram, 2026-05-13]

## Livis 建议的执行顺序

**② → ① → ③ → ④**

理由：需求 2（GBrain 认知调用）是基础设施。先搞清楚 Agent 怎么用 GBrain 的认知，后面做其他事效率更高、验证更清晰。

Henry 同意此排序。

[Source: User, Telegram, 2026-05-13]

## 背景

需求 2 直接对应 Henry 此前提出的核心问题："GBrain 里的认知如何被 Agent 在对话中真正调用起来？" 这是目前 brain 从"存储"升级到"使用"的关键卡点。
